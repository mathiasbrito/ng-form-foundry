New in 0.3.0

  • Validators + inline errors — hostname pattern/length, email & URI formats, integer port with min/max, multipleOf CPU share.
  • Optional shapes — Replicas is nullable; Notes is a presence toggle (absent until on); API version is a read-only constant.
  • Choice — labeled cases; the active case is inferred from the seed data (no __case was supplied).
  • Maps — open, arbitrary-keyed records you can add, remove, and rename.

Round-trip fixes it also ships

R1 — non-string map keys The Port map uses numeric keys (80, 443). They stay intact in the value above; the YAML/JSON transformers now reconcile such keys on write-back instead of appending stringified duplicates on a round-trip. R2 — integers beyond 2^53 Ledger id holds 9007199254740993 — past the safe integer range. Carried as a string, it keeps full precision (a JS number would round it to …992); the transformers re-emit it verbatim as an unquoted number. R3 — YANG identityref module A derived identity is now resolved by module, not first-name match, so a local-name collision (modB:turbo) keeps its original module on write-back. Exercised in the transformer test suite.
