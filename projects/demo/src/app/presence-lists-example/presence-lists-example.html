Optional (presence) lists

The affordance for a list follows whether it is required or optional (presence: true, i.e. advisoryRequired). The loaded config had none of the optional lists, so they start absent — nothing in the serialized value on the right. This is the byte-exact fix: a zero-edit rebuild never injects a list the source never authored.

  • Optional (amf_ip_address, plmn_list): an "Add …" button while absent → clicking it makes the first entry appear at once. Removing the last entry de-materializes the whole list (→ absent, key gone).
  • Required (tac_list): always present — no remove. When empty it stays [], with an "Add TAC item" button.
  • served_cells is an ordinary (non-presence) group list: always present, its section shown by default.
